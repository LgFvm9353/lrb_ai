# <></> 是什么？
- 解决了哪些问题？
  - jsx 最外层一定要有唯一的父元素
  - 不要为了div 而div

- 是什么？
  React中，<></>是一种语法糖，是Fragment组件的简写。它可以用来包裹一组元素，而不需要额外的父元素。

- 功能：
  避免多余的DOM 结构层次和元素
  性能更好

- 项目中一定要安排<Fragment >
  虚拟DOM层面 ：
    - Fragment在虚拟DOM中作为一个特殊的节点类型存在
    - 它不会对应到实际的DOM节点，只作为逻辑容器
  渲染机制 ：
    - React在协调(reconciliation)阶段会跳过Fragment节点
    - 直接处理其子元素，不会创建对应的DOM节点
