## 性能优化hook

- 父子组件渲染顺序
  - 执行的顺序：父组件 -》 子组件    （先外后内）
  - 渲染的顺序：子组件 -》 父组件    （先内后外）

- Button 组件不应该重新渲染
  - 原因：Button 组件没有使用 React.memo 进行优化
  - 解决方法：使用 React.memo 进行优化
  - 优化后：Button 组件不会重新渲染

  - 性能优化
    响应式和性能 非常好
    切分组件
    组件之间独立
    子组件 React.memo 
    createContext useContext 所有的状态全部给一个Context 好吗？
    不好，会造成性能优化的问题  所有状态都是通过一个reducer 生成

- 组件划分的粒度
  - 组件划分的粒度：组件的粒度越小，组件的复用性越高
  - 组件拆分，单向数据流
  - 就负责渲染的子组件 （props + jsx）
  - 复用，好管理之外，还可以提升性能
  - 状态更新后组件函数要重新运行
  - useCallback + React.memo
